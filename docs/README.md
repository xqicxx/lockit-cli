# Lockit Docs

这组文档用于把 `lockit` 从“已有雏形的 CLI 工具”推进成“可跨端、可同步、可被 agent 安全消费的凭据系统”。

## 目录分类

- `overview/`
  - `product-plan.md`：产品目标、范围、目标用户、MVP 边界
- `architecture/`
  - `system-architecture.md`：模块拆分、数据流、同步与导入能力设计
- `roadmap/`
  - `roadmap-todo.md`：分阶段路线图、优先级、待办列表
- `standards/`
  - `code-acceptance.md`：代码质量标准、测试门槛、验收清单

## 使用建议
- 先读 `overview/product-plan.md`，统一目标和边界。
- 再读 `architecture/system-architecture.md`，确认模块职责和接口。
- 进入开发前按 `roadmap/roadmap-todo.md` 拆任务。
- 提交实现前对照 `standards/code-acceptance.md` 做验收。
